Network & Forwarding

Access Control Networks ?IP ranges allowed to query this DNS server, in CIDR notation (e.g. 192.168.0.0/16).
Forward Servers ?Upstream DNS servers to forward queries to. Leave empty to use full recursive resolution (recommended for privacy).
DNS-over-TLS ?Encrypt forwarded queries using TLS (e.g. 1.1.1.1@853). Requires forward servers to be configured.
IPv4 ?Enable IPv4 for outgoing DNS queries to upstream servers.
IPv6 ?Enable IPv6 for outgoing DNS queries to upstream servers.
Prefer IPv4 ?Prefer IPv4 addresses over IPv6 when resolving DNS queries.

Performance

Threads ?Number of threads for processing DNS queries. Match this to your CPU core count for best performance. restart required
Prefetch ?Prefetch popular cached entries before they expire, reducing latency for frequently accessed domains.
Fast Server Permil ?How often to prefer the fastest upstream server (0-1000, in per mille). Higher values favor speed over load balancing.
Fast Server Count ?Number of fastest upstream servers to track and prefer.
EDNS Buffer Size ?EDNS reassembly buffer size in bytes. Default 1232 avoids fragmentation on most networks. Increase to 4096 only if needed.
Minimal Responses ?Only include necessary data in DNS responses. Saves bandwidth and slightly improves performance.

Cache Controls

Cache Sizing

Message Cache (MB) ?Size of the message cache. Stores DNS response metadata. Increase for better hit rates on busy networks.
RRset Cache (MB) ?Size of the RRset (resource record set) cache. Stores actual DNS record data. Recommended to be roughly 2x message cache size.
Negative Cache (MB) ?Size of the cache for negative responses (NXDOMAIN, NODATA). Prevents repeated lookups for non-existent domains.

TTL & Expiry

Min TTL (seconds) ?Minimum time-to-live for cached records. Increasing this reduces upstream queries but may serve slightly stale data.
Max TTL (seconds) ?Maximum time-to-live for cached records. Caps how long records stay cached regardless of their original TTL.
Max Negative TTL (s) ?Maximum TTL for negative responses (NXDOMAIN). Lower values mean faster recovery when a new domain is created.
Serve Expired ?Serve stale cache entries while fetching fresh data in the background. Greatly improves perceived DNS speed and resilience to upstream outages.
Serve Expired TTL (s) ?How long expired entries can still be served while a refresh is in progress. Default 86400 (1 day). Only applies when Serve Expired is enabled.
Aggressive NSEC ?Use cached NSEC/NSEC3 records to prove that domains don't exist without extra queries. Free performance win when DNSSEC is enabled.

Security & Privacy

DNSSEC Validation ?Verify that DNS responses are authentic and haven't been tampered with using cryptographic signatures.
QNAME Minimisation ?Send only the minimum necessary part of the domain name to each upstream server during resolution. Improves privacy.
Hide Identity ?Refuse to respond to queries for the server's identity (id.server, hostname.bind). Prevents fingerprinting.
Hide Version ?Refuse to respond to queries for the server's version (version.server, version.bind). Prevents fingerprinting.
Use CAPS for ID ?Randomize letter case in outgoing queries (0x20 encoding). Adds extra protection against DNS spoofing by using case as an additional verification channel.

Custom Config

Use custom unbound.confrestart required

When enabled, place your config at the addon's /config mount (host path: /addon_configs/<slug>_unbound/unbound.conf). All other settings are ignored.

Logging

Verbosity ?Logging detail level: 0 = errors only, 1 = operational info, 2 = detailed, 3-5 = debug levels.
Log Queries ?Log all DNS queries. Required for the Query Log tab to work. May impact performance on busy networks.
Log Replies ?Log DNS replies (response code, latency). Independent of Log Queries. Enabling both will record each lookup twice in the query log.
